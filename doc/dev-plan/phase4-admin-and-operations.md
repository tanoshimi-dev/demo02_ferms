# Phase 4 - 管理運用機能

## Goal

管理者が施設・設備・予約を運用できる状態を作り、FERMS を日常運用可能な MVP にする。

## Scope

- 管理者認可
- 施設管理
- 設備管理
- 予約管理
- 公開状態 / 利用可否 / 基本運用設定
- 管理画面と管理 API のテストコード

## Tasks

1. 管理者ロール判定を実装する
2. 施設登録 / 更新 / 無効化 API を実装する
3. 設備登録 / 更新 / 無効化 API を実装する
4. 予約検索 / 状態変更 / 管理キャンセル API を実装する
5. 管理画面用の一覧と編集フォームを作る
6. 公開状態や利用可否の変更が利用者画面へ反映されるようにつなぐ
7. 必要最低限の監査観点を整理する
8. 管理 API とロール判定のテストコードを追加する
9. 管理画面の主要操作に対するテストコードを追加する
10. 起動した frontend / backend / db に対して管理運用シナリオのテストを実施する

## Deliverables

- 管理者専用 API
- 施設管理画面
- 設備管理画面
- 予約管理画面
- 管理運用テストコード

## Done

- 管理者以外は管理 API に入れない
- 施設 / 設備を運用状態込みで管理できる
- 予約状況を検索し、管理都合で調整できる
- 利用者導線と管理者導線が矛盾しない
- 起動した対象サービスに対して管理運用テストを実行できる

## Notes

- 認可は backend 側で最終制御する
- role の表現は portal claims と将来整合する形にする
- MVP の運用状態切り替えは `isActive` を基本の制御項目として扱う
- ユーザー管理は基本枠に留め、MVP では深追いしない
- 管理設定変更後は利用者導線側の影響も起動中サービスで確認する
